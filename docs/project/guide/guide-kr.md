# 가상 메모리 구조 가이드

## 현재 구조

```
+----------------------------------+
|           index.js               |  <- 진입점
+----------------------------------+
|        elementManager            |  <- 단일 요소 (싱글톤)
+----------------------------------+
|        elementService            |  <- 순수 함수 (상태 없음)
+----------------------------------+
|          element                 |  <- 데이터 구조 (리터럴)
+----------------------------------+
```

### 현재 한계점

| 문제 | 설명 |
|------|------|
| 단일 객체만 가능 | elementManager가 하나의 element만 보유 |
| 캐시 없음 | 매번 새로 생성 |
| 메모리 관리 없음 | GC 개념 없음 |

---

## 목표 구조: VirtualMemory 레이어 추가

```
+----------------------------------+
|           index.js               |  <- 진입점
+----------------------------------+
|        VirtualMemory             |  <- [신규] Pool + Cache + GC
+----------------------------------+
|        elementManager            |  <- Factory 패턴으로 변경
+----------------------------------+
|        elementService            |  <- 유지 (순수 함수)
+----------------------------------+
|          element                 |  <- 유지 (데이터 구조)
+----------------------------------+
```

---

## 새 레이어 상세

### 1. VirtualMemory

**역할**: 최상위 메모리 관리자

**책임**:
- 노드 풀 관리
- 캐시 관리
- 가비지 컬렉션 (GC)

**구조**:
```
VirtualMemory
|
+-- pool: {}           <- 모든 노드 저장소 (키: 주소)
+-- cache: {}          <- 자주 사용되는 노드
+-- cacheOrder: []     <- LRU 순서 추적
+-- maxCacheSize: n    <- 최대 캐시 크기
|
+-- allocate()         <- 새 메모리 할당 (노드 생성)
+-- free(address)      <- 메모리 해제
+-- read(address)      <- 메모리 읽기 (캐시 우선)
+-- write(address, v)  <- 메모리 쓰기
+-- gc()               <- 가비지 컬렉션 실행
```

### 2. 캐시 레이어

**역할**: 자주 접근하는 노드에 대한 빠른 조회

**전략**: LRU (Least Recently Used, 가장 오래 전에 사용된 항목 제거)

```
+--------------------------------------------------+
|                  LRU 캐시 흐름                     |
+--------------------------------------------------+
|  1. read(addr) 호출                               |
|  2. 캐시에 있으면 -> 반환 + 순서 갱신              |
|  3. 캐시에 없으면 -> pool에서 가져옴 + 캐시 추가    |
|  4. 캐시가 가득 차면 -> 가장 오래된 항목 제거       |
+--------------------------------------------------+
```

### 3. Node (메모리 블록)

**역할**: 메모리 블록으로 확장된 element

**구조**:
```
Node {
    address: "0x001"    <- 고유 주소 (기존 key 역할)
    value: any          <- 저장된 값
    refCount: 0         <- 참조 카운트 (GC용)
    createdAt: timestamp
    lastAccess: timestamp
}
```

## ES5 메모리 관리 방법

| 방법 | 설명 | ES5 지원 |
|------|------|----------|
| Object | 키-값 저장소 | O |
| Array | 순서 추적 (LRU) | O |
| WeakMap | GC 친화적 참조 | X (ES6) |
| Closure | 비공개 상태 | O |

**ES5 최적 방법**: Object + Array + Closure

```
+--------------------------------------------------+
|              ES5 메모리 패턴                       |
+--------------------------------------------------+
|  var pool = {};         <- 노드 저장소            |
|  var cache = {};        <- 캐시 저장소            |
|  var order = [];        <- LRU 순서               |
|                                                   |
|  Closure로 비공개 상태 관리                        |
|  refCount로 GC 대상 탐지                          |
+--------------------------------------------------+
```

```
예시:
- 0x17328456789_a1b2
- 0x17328456790_c3d4

+--------------------------------------------------+
|                 주소 구조                          |
+--------------------------------------------------+
|  0x       | timestamp     | _ | random           |
|  접두사    | 생성 시간      |   | 충돌 방지        |
+--------------------------------------------------+
```

### VirtualMemory 개념 분해

**VirtualMemory**는 단일 객체가 아닌 **여러 레이어로 분리**됩니다:

```
+--------------------------------------------------+
|              VirtualMemory (전체 개념)            |
+--------------------------------------------------+
|                                                   |
|  +-- memoryRepo.js    <- Pool 관리 (저장소)       |
|  +-- cacheRepo.js     <- Cache 관리 (LRU)        |
|  +-- gcService.js     <- GC 로직 (정리)          |
|                                                   |
+--------------------------------------------------+
```

**역할 매핑:**

| 파일 | VirtualMemory 기능 | 역할 |
|------|-------------------|------|
| `memoryRepo.js` | pool | 노드 저장/조회/삭제 (CRUD) |
| `cacheRepo.js` | cache | LRU 캐시, 빠른 조회 |
| `gcService.js` | gc() | refCount 기반 메모리 정리 |

**Heap 관점 비교:**

```
+--------------------------------------------------+
|                    JS Heap                        |
+--------------------------------------------------+
|                                                   |
|  [기존 guide.md 설계]                             |
|  VirtualMemory (단일 객체)                        |
|  +-- pool: {}                                    |
|  +-- cache: {}                                   |
|  +-- gc()                                        |
|                                                   |
+--------------------------------------------------+
|                                                   |
|  [실무 패턴 적용 후]                              |
|  memoryRepo (0xMR001)  <- pool 담당              |
|  +-- pool: {}                                    |
|  +-- allocate()                                  |
|  +-- free()                                      |
|                                                   |
|  cacheRepo (0xCR001)   <- cache 담당             |
|  +-- cache: {}                                   |
|  +-- order: []                                   |
|  +-- get()                                       |
|  +-- set()                                       |
|                                                   |
|  gcService             <- gc 로직 담당           |
|  +-- collect()                                   |
|                                                   |
+--------------------------------------------------+
```
