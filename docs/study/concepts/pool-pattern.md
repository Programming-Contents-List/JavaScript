# Pool 패턴

## Pool이란?

**정의:** 미리 생성해둔 리소스(객체)를 재사용하는 패턴

```
+--------------------------------------------------+
|              Pool 개념                            |
+--------------------------------------------------+
|                                                   |
|  일반적인 방식 (Pool 없음)                         |
|  +-- 필요할 때마다 new 생성                       |
|  +-- 사용 후 GC가 수거                            |
|  +-- 반복 시 생성/소멸 비용 발생                   |
|                                                   |
|  Pool 방식                                        |
|  +-- 미리 객체를 생성해둠                         |
|  +-- 필요할 때 Pool에서 꺼내 사용                  |
|  +-- 사용 후 Pool에 반납                          |
|  +-- 생성/소멸 비용 절감                          |
|                                                   |
+--------------------------------------------------+
```

---

## 왜 사용하는가?

### 문제 상황

```
+--------------------------------------------------+
|              객체 생성 비용                        |
+--------------------------------------------------+
|                                                   |
|  [반복 생성/소멸]                                 |
|  for (var i = 0; i < 1000; i++) {                |
|      var obj = new Object();  // 생성 비용        |
|      // 사용                                      |
|      // GC 대상                  // 소멸 비용     |
|  }                                               |
|                                                   |
|  총 비용: 생성 1000번 + GC 1000번                 |
|                                                   |
+--------------------------------------------------+
```

### Pool 적용 시

```
+--------------------------------------------------+
|              Pool 재사용                          |
+--------------------------------------------------+
|                                                   |
|  [초기화]                                         |
|  var pool = [obj1, obj2, obj3, ...];  // 1회 생성|
|                                                   |
|  [사용]                                           |
|  var obj = pool.pop();   // 꺼내기               |
|  // 사용                                          |
|  pool.push(obj);         // 반납                 |
|                                                   |
|  총 비용: 생성 N번 (초기화 시) + GC 0번           |
|                                                   |
+--------------------------------------------------+
```

---

## Pool의 종류

| 종류 | 설명 | 사용처 |
|------|------|--------|
| Object Pool | 일반 객체 재사용 | 게임, 파티클 |
| Connection Pool | DB 연결 재사용 | 서버, 백엔드 |
| Thread Pool | 스레드 재사용 | 멀티스레딩 |
| Memory Pool | 메모리 블록 재사용 | 시스템 프로그래밍 |

## Heap 관점에서 이해

### Pool 없이 (매번 생성)

```
+-------------------------+-------------------------+
|         Stack           |          Heap           |
+-------------------------+-------------------------+
|                         |                         |
|  [1회차]                |  obj1 (0x001) ← 생성    |
|  obj: 0x001 ----------->|                         |
|                         |                         |
|  [2회차]                |  obj1 (0x001) ← GC 대상 |
|  obj: 0x002 ----------->|  obj2 (0x002) ← 생성    |
|                         |                         |
|  [3회차]                |  obj2 (0x002) ← GC 대상 |
|  obj: 0x003 ----------->|  obj3 (0x003) ← 생성    |
|                         |                         |
+-------------------------+-------------------------+
|  문제: 매번 새 주소 할당, GC 부담 증가             |
+--------------------------------------------------+
```

### Pool 사용 시 (재사용)

```
+-------------------------+-------------------------+
|         Stack           |          Heap           |
+-------------------------+-------------------------+
|                         |                         |
|  pool: 0xP001 --------->|  Pool (0xP001)          |
|                         |  +-- [0]: obj (0x001)   |
|                         |  +-- [1]: obj (0x002)   |
|                         |  +-- [2]: obj (0x003)   |
|                         |                         |
|  [1회차 - 꺼내기]        |                         |
|  obj: 0x001 ----------->|  obj (0x001) ← 재사용   |
|                         |                         |
|  [1회차 - 반납]          |                         |
|  pool[0] = obj -------->|  Pool에 다시 저장       |
|                         |                         |
+-------------------------+-------------------------+
|  장점: 동일 주소 재사용, GC 부담 감소              |
+--------------------------------------------------+
```

## Pool vs Cache 차이

| 항목 | Pool | Cache |
|------|------|-------|
| 목적 | 객체 **재사용** | 데이터 **빠른 접근** |
| 저장 대상 | 빈 객체 (재사용 대기) | 값이 있는 데이터 |
| 꺼낸 후 | Pool에서 **제거** | Cache에 **유지** |
| 반납 | 사용 후 **반납 필요** | 반납 개념 없음 |
| 알고리즘 | LIFO/FIFO | LRU, LFU 등 |

```
+--------------------------------------------------+
|              Pool vs Cache 흐름                   |
+--------------------------------------------------+
|                                                   |
|  [Pool]                                          |
|  꺼내기: pool.pop() → 객체 반환, pool에서 제거    |
|  반납:   pool.push(obj) → 다시 pool에 저장       |
|                                                   |
|  [Cache]                                         |
|  조회:   cache.get(key) → 값 반환, cache 유지    |
|  저장:   cache.set(key, val) → cache에 저장      |
|                                                   |
+--------------------------------------------------+
```

## 현재 프로젝트에서의 Pool

### memoryRepo의 pool

```
+--------------------------------------------------+
|              memoryRepo.pool의 역할               |
+--------------------------------------------------+
|                                                   |
|  일반적인 Pool: 빈 객체 재사용                    |
|  memoryRepo.pool: node 저장소 (약간 다름)         |
|                                                   |
|  차이점:                                          |
|  +-- 일반 Pool: 빈 객체를 꺼내서 채움             |
|  +-- memoryRepo: 값이 있는 node를 저장/관리       |
|                                                   |
|  공통점:                                          |
|  +-- 중앙 집중식 저장소                           |
|  +-- 리소스(node) 관리                           |
|                                                   |
+--------------------------------------------------+
```

### 구조

```
+--------------------------------------------------+
|              memoryRepo.pool 구조                 |
+--------------------------------------------------+
|                                                   |
|  pool = {                                        |
|      "0x001": node1,   ← address가 key           |
|      "0x002": node2,                             |
|      "0x003": node3                              |
|  }                                               |
|                                                   |
|  자료구조: Object (key-value)                    |
|  key: node의 address                             |
|  value: node 객체                                |
|                                                   |
+--------------------------------------------------+
```

## 디자인 패턴

**적용 패턴:** Object Pool Pattern

**SOLID 관점:**
- **S**RP: Pool은 객체 저장/관리만 담당
- **O**CP: 새로운 객체 타입 추가 시 Pool 수정 불필요
- **D**IP: 상위 모듈이 Pool 인터페이스에 의존

```
+--------------------------------------------------+
|              Pool 핵심 정리                       |
+--------------------------------------------------+
|                                                   |
|  Pool = 리소스 재사용을 위한 저장소               |
|                                                   |
|  장점:                                            |
|  +-- 생성/소멸 비용 절감                          |
|  +-- GC 부담 감소                                |
|  +-- 메모리 효율성 향상                          |
|                                                   |
|  memoryRepo.pool:                                |
|  +-- node를 저장하는 중앙 저장소                  |
|  +-- address를 key로 사용                        |
|  +-- CRUD 메서드로 접근                          |
|                                                   |
+--------------------------------------------------+
```
