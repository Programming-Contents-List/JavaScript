# IIFE와 Closure 개념 가이드

## IIFE (Immediately Invoked Function Expression)

**즉시 실행 함수 표현식** - 정의와 동시에 실행되는 함수

---

## 기본 문법

```js
// 기본 형태
(function() {
    // 코드 실행
})();

// 반환값 할당
var result = (function() {
    return "Hello";
})();
```

---

## IIFE의 목적

```
+--------------------------------------------------+
|                 IIFE 사용 목적                     |
+--------------------------------------------------+
|  1. 전역 스코프 오염 방지                          |
|  2. 비공개 변수 생성 (Closure 활용)                |
|  3. 모듈 패턴 구현 (ES5)                          |
|  4. 초기화 코드 캡슐화                             |
+--------------------------------------------------+
```

---

## Closure 유무에 따른 차이

### Case 1: Closure 없이 (단순 IIFE)

```js
var element = (function() {
    return {
        key: "",
        value: ""
    };
})();

// 사용
element.key = "test";      // ✅ 직접 수정 가능
element.value = "hello";   // ✅ 직접 수정 가능
console.log(element.key);  // "test"
```

**특징:**
- 객체 프로퍼티 직접 접근/수정 가능
- 캡슐화 효과 없음
- 단순히 즉시 실행만 함

---

### Case 2: Closure 사용 (캡슐화)

```js
var element = (function() {
    // 비공개 변수 (Closure로 보호)
    var _key = "";
    var _value = "";

    // 공개 인터페이스 반환
    return {
        getKey: function() { return _key; },
        setKey: function(k) { _key = k; },
        getValue: function() { return _value; },
        setValue: function(v) { _value = v; }
    };
})();

// 사용
element._key;              // ❌ undefined (접근 불가)
element.setKey("test");    // ✅ 메서드로 수정
element.getKey();          // ✅ "test" (메서드로 조회)
```

**특징:**
- 내부 변수 외부 접근 불가
- getter/setter로만 제어
- 진정한 캡슐화

## Heap과 Stack 관점

### Stack과 Heap 기본 개념

```
+--------------------------------------------------+
|                메모리 구조                         |
+--------------------------------------------------+
|                                                   |
|  Stack (스택)                                     |
|  +-- 원시값 저장 (number, string, boolean)        |
|  +-- 함수 호출 정보 (Call Stack)                  |
|  +-- 참조 주소 저장                               |
|  +-- LIFO (Last In First Out)                    |
|                                                   |
|  Heap (힙)                                        |
|  +-- 객체 저장 (Object, Array, Function)          |
|  +-- 동적 메모리 할당                             |
|  +-- 참조로 접근                                  |
|                                                   |
+--------------------------------------------------+
```

---

### Case 1: Closure 없이 - 메모리 상태

```js
var element = (function() {
    return { key: "", value: "" };
})();
```

**실행 단계:**

```
+--------------------------------------------------+
|              단계 1: IIFE 실행 전                  |
+--------------------------------------------------+
|  Stack                    |  Heap                |
|---------------------------|----------------------|
|  element: undefined       |                      |
+--------------------------------------------------+

+--------------------------------------------------+
|              단계 2: IIFE 실행 중                  |
+--------------------------------------------------+
|  Stack                    |  Heap                |
|---------------------------|----------------------|
|  [IIFE Call Frame]        |  익명함수 (0xF001)   |
|  +-- (실행 컨텍스트)       |                      |
+--------------------------------------------------+

+--------------------------------------------------+
|              단계 3: IIFE 실행 후                  |
+--------------------------------------------------+
|  Stack                    |  Heap                |
|---------------------------|----------------------|
|  element: 0xE001 -------->|  Object (0xE001)     |
|                           |  +-- key: ""         |
|                           |  +-- value: ""       |
|                           |                      |
|                           |  익명함수 (0xF001)   |
|                           |  └─ GC 대상 (참조X)  |
+--------------------------------------------------+
```

**핵심:**
- IIFE 실행 후 익명 함수는 GC 대상
- 반환된 객체만 Heap에 남음
- 객체에 직접 접근 가능

---

### Case 2: Closure 사용 - 메모리 상태

```js
var element = (function() {
    var _key = "";
    var _value = "";

    return {
        getKey: function() { return _key; },
        setKey: function(k) { _key = k; }
    };
})();
```

**실행 단계:**

```
+--------------------------------------------------+
|              단계 1: IIFE 실행 전                  |
+--------------------------------------------------+
|  Stack                    |  Heap                |
|---------------------------|----------------------|
|  element: undefined       |                      |
+--------------------------------------------------+

+--------------------------------------------------+
|              단계 2: IIFE 실행 중                  |
+--------------------------------------------------+
|  Stack                    |  Heap                |
|---------------------------|----------------------|
|  [IIFE Call Frame]        |  익명함수 (0xF001)   |
|  +-- _key: ""             |                      |
|  +-- _value: ""           |                      |
+--------------------------------------------------+

+--------------------------------------------------+
|              단계 3: IIFE 실행 후 (Closure 생성)   |
+--------------------------------------------------+
|  Stack                    |  Heap                |
|---------------------------|----------------------|
|  element: 0xE001 -------->|  Object (0xE001)     |
|                           |  +-- getKey: 0xG001  |
|                           |  +-- setKey: 0xS001  |
|                           |                      |
|                           |  Closure (0xC001)    |
|                           |  +-- _key: ""        |
|                           |  +-- _value: ""      |
|                           |       ^              |
|                           |       |              |
|                           |  getKey (0xG001) ----+
|                           |  setKey (0xS001) ----+
|                           |                      |
|                           |  익명함수 (0xF001)   |
|                           |  └─ GC 대상 (참조X)  |
+--------------------------------------------------+
```

**핵심:**
- **Closure Scope**가 Heap에 유지됨
- `_key`, `_value`는 Closure 내부에 존재
- `getKey`, `setKey` 함수가 Closure를 참조
- 외부에서 `_key`, `_value` 직접 접근 불가

---

## Closure가 GC되지 않는 이유

```
+--------------------------------------------------+
|              참조 체인 (Reference Chain)           |
+--------------------------------------------------+
|                                                   |
|  전역                                             |
|    |                                              |
|    v                                              |
|  element (0xE001)                                 |
|    |                                              |
|    +-- getKey (0xG001)                            |
|    |     |                                        |
|    |     +---> [[Scope]]: Closure (0xC001)        |
|    |                        |                     |
|    |                        +-- _key              |
|    |                        +-- _value            |
|    |                                              |
|    +-- setKey (0xS001)                            |
|          |                                        |
|          +---> [[Scope]]: Closure (0xC001)        |
|                                                   |
+--------------------------------------------------+
|  결과: element → 함수들 → Closure 참조 유지        |
|        → Closure는 GC되지 않음                    |
+--------------------------------------------------+
```

## 비교 요약

```
+--------------------------------------------------+
|           Closure 유무 비교 정리                   |
+--------------------------------------------------+
|  항목              | Closure 없음 | Closure 있음  |
+--------------------------------------------------+
|  외부 접근         |     ✅       |      ❌       |
|  직접 수정         |     ✅       |      ❌       |
|  캡슐화            |     ❌       |      ✅       |
|  메모리 사용       |     적음     |      많음     |
|  Closure Scope    |     없음     |      있음     |
|  GC 대상          |  함수만 GC   | Closure 유지  |
+--------------------------------------------------+
```

## 실전 예시: 카운터

### Closure 없이 (문제점)

```js
var counter = (function() {
    return {
        count: 0,
        increment: function() { this.count++; },
        getCount: function() { return this.count; }
    };
})();

counter.count = 9999;  // ❌ 외부에서 직접 조작 가능!
counter.getCount();    // 9999
```

### Closure 사용 (안전)

```js
var counter = (function() {
    var _count = 0;  // 비공개

    return {
        increment: function() { _count++; },
        decrement: function() { _count--; },
        getCount: function() { return _count; }
    };
})();

counter._count;        // ❌ undefined
counter._count = 9999; // ❌ 새 프로퍼티 생성될 뿐, 내부 _count 영향 없음
counter.getCount();    // 0 (안전하게 유지)
counter.increment();
counter.getCount();    // 1
```

## Heap 뷰: 카운터 예시

```
+--------------------------------------------------+
|                    JS Heap                        |
+--------------------------------------------------+
|                                                   |
|  counter (0xCT001)                               |
|  +-- increment: function (0xI001) ─┐             |
|  +-- decrement: function (0xD001) ─┼─┐           |
|  +-- getCount:  function (0xG001) ─┼─┼─┐         |
|                                    │ │ │         |
|  Closure Scope (0xCS001) <─────────┴─┴─┘         |
|  +-- _count: 1                                   |
|       ↑                                          |
|       └─ 외부 접근 불가 (비공개)                  |
|                                                   |
+--------------------------------------------------+
```

## 디자인 패턴

**적용 패턴:** Module Pattern (모듈 패턴)

- IIFE + Closure를 조합한 ES5 모듈화 기법
- 비공개 상태와 공개 인터페이스 분리
- ES6 `export/import` 이전의 표준 패턴

---

## 정리

| 상황 | 권장 방식 |
|------|----------|
| 단순 데이터 구조 | Closure 없이 |
| 상태 보호 필요 | Closure 사용 |
| 메모리 최적화 필요 | Closure 없이 |
| 캡슐화/정보 은닉 | Closure 사용 |
