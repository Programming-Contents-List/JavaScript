# Section2-3의 Layer와 관심사 분리

> 어떻게 하면 js를 OOP 답게 코드를 작성할 수 있을가 부터 시작을 했다. 그래서 나는 실무에서 얻었던 Layer 방식을 사용할까 한다.

## Layer

실무에서는 ts 환경으로 개발을 하고 작업을 한다. 그래서 object까지 도달하면서 내가 관찰한 Layer들을 설명하면 다음과 같다. (가장 큰 덩어리 순으로 내려오면 다음과 같다.)

```md
1. View(page - ex: Login)
2. Hook(custom functionality)
3. Service
4. Mannager
5. Object(type)
```

이러한 순서대로 작업이 진행되면서 네이밍 컨벤션 규칙들이 이루어진다. section2-3에서도 위와 같은 Layer를 사용해서 작업이 이루어질 예정이다. 단, 다음과 같은 고민들에 빠지게된다.

1. FSD 패턴과 같은 큰 덩이러리를 다루는 패턴들은 어떻게 되는 것일까?
2. 객체기반 철학의 FP를 사용하기 위해서는 창의성을 발휘 해야하는데 우리는 어떻게 창의성을 발휘 하는가.
3. 패턴들의 유기적인 관계성의 근간과 근본은 무엇인가. 비즈니스 모델에 따른 관계와 형태인가.

이러한 고민을 하지만, 위 세가지의 질문은 다음 section에서 이루어질 과제와 다름 없다.
