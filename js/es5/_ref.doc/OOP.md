# 객체지향이란

Js는 프로토타입 기반으로 구현되어 있는 언어로, 클래스가 없다. 따라서, 한국 서적에 있는 객체지향 언어는 프로토타입 기반 언어의 객체지향을 이해하기에 어려움이 있다. 하지만, 객체지향에서 지향하고 있는 몇가지 추상화들만 쫒아간다면, Js를 객체지향으로 풀어서 개념을 잡아 낼 수 있지 않을까 싶다.

## 책임과 역할

리터럴은 왜 존재할까? 리터럴은 프로퍼티를 담을 수 있다. 그렇다면 프로퍼티는 Js에서 어떤 의미를 지니고 있을까? Js에서 프로퍼티의 직역은 속성이지만 문법적으로 객체의 상태를 구성하는 의미하며 Js 프로퍼티의 특징이다.
