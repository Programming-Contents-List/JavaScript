# Heap 이란?

Js는 C,C++과 같이 heap 메모리에 접근을 해서 메모리 주소에 접근할 수 없다. js는 메모리를 안전하게 관리하는 메모리 안전 언어(memory safe language)이기 때문이다. 직접적인 주소를 생성하고 관리하는 방식을 고안하던 중, weakmap과 map의 활용을 고민했으나. 고유햔 아이디를 부여하고 생성된 객체의 프로퍼티를 식별하기에 어려움이 있어서 uuid를 사용하는 것이 최선으로 보이며, 필요한 개념은 클로저와 IFFE개념과 싱글톤패턴이 있다. 차라리, heap에 접근이 어려우니 cache를 고안하려고 한다.

- heap은 물리적 저장소, 주소 기반
- cache는 논리적 저장소, heap을 참조. key 기반(cache는 Heap의 목차)

## Heap 동작 원리

Heap은 완전 이진 트리 구조를 사용해서 동작한다. 이로 인해 최소값, 최대값을 효율적으로 찾기위함이다. Browser의 DOM역시 트리 구조이다. 그렇다면, Heap이냐는 질문에는 그렇지 않다. 그저 tree의 구조를 가지고 있을 뿐, 동작은 그렇지 못하다. 물론, Heap과 같이 동작할 수 있게 할 수는 있으나 비효율적이고 tree를 재구성해야하는 리스크와 Heap을 사용한다는 것은 DOM이라는 사용목적에 맞지 않는 방향이다.
