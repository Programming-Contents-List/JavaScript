# Factory(팩토리)

팩토리는 객체를 생성하는 함수를 의미한다. 즉, 객체 생성 과정을 함수로 은닉하는 패턴이다. 정리하면, new 없이 객체를 만들어서 반환하는 함수이다.

## 특징

호출 마다 새로운 객체를 생성한다. 단, new를 사용하면, 생성자 기반이며, prototype을 공유하게 된다.

## 주의

팩토리와 팩토리 패턴은 다른 개념이다.

### 그렇다면 캡슐과 팩토리는 다른 개념인가?

- 팩토리는 "생성 책임을 묶는 방식"이다.
- 캡슐화는 "내부 상태를 숨기고(또는 통제하고) 외부엔 필요한 인터페이스만 노출"하는 것이다.
- 결론적으로 팩토리는 공개 프로퍼티가 되기 때문에 캡슐화라고 보기에는 어렵다.
- 만일, 객체 리터럴로 캡슐화를 하기 위해서는 ES5문법으로 Object 문법을 강화할 필요가 있다.
- 캡슐화를 하기 위해서는 읽기 전용과 쓰기 전용으로 나뉘어서 생각해보면 도움이 될 듯 하다.
