# js 객체들의 선회

> Js는 prototype(프로토타입) 기반 객체이다. 우리가 말하는 함수형은 아래의 리스트를 모두 통틀어서 말한다. 하지만, Js에서 객체 생성은 결국 함수와 prototype chain으로 연결된다.

## 객체 리터럴 방식

- 즉석 객체, 단발성

## 객체 함수형 생성자(= fn.prototype) 방식

> 여기서 prototype 개념이 깊게 들어가 있다. 단, ES5를 기준으로 작성해보는 것을 권장한다.

- ES5, new + prototype

## 객체 함수형 방식(Pure Function)

> 여기서 JS 함수형이란, 순수한 함수를 의미하며, 우리가 난발하고 사용하고 있는 JS 함수형(Functional Programming)인 함수들은 `객체 생성/조합 스타일`이라고 봐야 한다.

- 순수 함수는 JS 함수형(합성·재사용) 유리

- 순수함수는 리터럴에 붙을 수 있고 함수에게도 붙을 수 있으며, 생성자에 붙어가며 우리가 알고 있는 JS 함수형(Functional Programming)에 가까워진다.
