# dir introduction

> 디렉토리 구조를 설명하는 문서.

## ~/`<dir>`/memory

- 해당 디렉토리는 C,C++과 같이 heap 메모리에 접근을 해서 메모리 주소에 접근하려 했던, 시도들이 담겨 있다. 하지만, js는 메모리를 안전하게 관리하는 언어로 메모리에 접근할 수 없다. 직접적인 주소를 생성하고 관리하는 방식을 고안하던 중, weakmap과 map의 활용을 고민했으나. 고유햔 아이디를 부여하고 생성된 객체의 프로퍼티를 식별하기에 어려움이 있어서 uuid를 사용하는 것이 최선으로 보이며, 필요한 개념은 클로저와 IFFE개념과 싱글톤패턴이 있다. 차라리, heap에 접근이 어려우니 cache를 고안하려고 한다.

- heap은 물리적 저장소, 주소 기반
- cache는 논리적 저장소, heap을 참조. key 기반(cache는 Heap의 목차)
