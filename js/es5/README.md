# js 객체들의 선회

> Js는 prototype(프로토타입) 기반 객체이다. 우리가 말하는 함수형은 아래의 리스트를 모두 통틀어서 말한다. 하지만, Js에서 객체 생성은 결국 함수와 prototype chain으로 연결된다.

## 객체 리터럴 방식

- 즉석 객체, 단발성
- 러터럴을 기준으로 함수를 만들어 낼 수도 있다. 이를 Method라 지칭한다. 리터럴을 기준으로 코드를 구성하게 된다면 굉장히 정적으로 동작하는 것으로 보인다. 즉, 리터럴 내부의 객체들을 외부로 부터 주입을 받기가 어렵다. 이를 동적으로 활용하기 위해서는 결국 함수를 활용하게 된다는 것이다.

## 객체 함수형 생성자(= fn.prototype) 방식

> 여기서 prototype 개념이 깊게 들어가 있다. 단, ES5를 기준으로 작성해보는 것을 권장한다.

- 함수 자체로 존재하는 것으로 이것을 독립 함수라 보고 함수형에 가깝다. 함수형으로 Js, React, 등 다양한 코드가 함수형으로 되어 있는 이유는 동적으로 메모리(힙)의 값을 변경 할 수 있다.

- ES5, new + prototype

## 객체 함수형 방식(Pure Function)

> 여기서 JS 함수형이란, 순수한 함수를 의미하며, 우리가 난발하고 사용하고 있는 JS 함수형(Functional Programming)인 함수들은 `객체 생성/조합 스타일`이라고 봐야 한다.

- 순수 함수는 JS 함수형(합성·재사용) 유리

- 순수함수는 리터럴에 붙을 수 있고 함수에게도 붙을 수 있으며, 생성자에 붙어가며 우리가 알고 있는 JS 함수형(Functional Programming)에 가까워진다.

### 변수는 무엇인가

- 변수는 JS에서 객체로 저장된 힙이란 저장소를 명확하게 구별짓기 위한 하나의 alias로 보면 좋다.

### 절차지향적인 함수형

- React에서 Fiber 코드를 살펴보면, 연결리스트로 구현이 되어있다.(물론 더 확인이 필요하다.) 물론, insert와 delete가 존재하지만 나열된 모습은 직렬로 이루어져 있다. 더 불어서 Nodejs는 싱글스레드이기 때문에 병렬적으로 동작할 수가 없다.
